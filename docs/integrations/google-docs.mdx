# Google Docs Integration via OpenInt

OpenInt enables your app to securely and programmatically connect with Google Docs, leveraging OAuth 2.0 for delegated user access. This guide details how to configure your Google Cloud project, set up the necessary credentials, and integrate Google Docs capabilities using OpenInt.

> ℹ️ This guide is for developers integrating Google Docs via OpenInt. For more about how OpenInt handles connectors, see [Core Concepts](/core-concepts-v0/connector-config).

## Integration Overview

- **Provider:** Google Docs
- **OpenInt Stage:** General Availability (GA)
- **OAuth protocol:** OAuth 2.0
- **Supported OpenInt Scope:**
    - `https://www.googleapis.com/auth/documents.currentonly` _(temporary access to the currently open Google Doc)_

Currently, OpenInt only supports the most limited Docs API scope. If you require additional scopes, please [contact OpenInt support](/connect/faq).

## Prerequisites

- A Google account
- Access to the [Google Cloud Console](https://console.developers.google.com/)
- Admin access to your OpenInt configuration (yours or your customers’)

## Step 1: Create a Google Cloud Project

1. Go to the [Google API Console](https://console.developers.google.com/).
2. Click on **Select a Project** and choose your existing project, or click **New Project** to create one.
3. Give your project a descriptive name (e.g., "OpenInt Google Docs Integration").

## Step 2: Enable Google Docs API

1. In the left sidebar, click **APIs & Services > Enabled APIs & services**.
2. Click **+ ENABLE APIS AND SERVICES** at the top.
3. Search for “Google Docs API” and click **Enable**.
   - If you need Google Drive file-level access (certain scopes), also enable “Google Drive API”.

## Step 3: Configure the OAuth Consent Screen

1. In your Google Cloud project, navigate to **APIs & Services > OAuth consent screen**.
2. Choose **External** and click **Create**.
3. Fill in required fields:
   - _App Name_, _User Support Email_, _Developer Contact Email_.
   - (You can skip app logo and other fields for development/testing.)
4. Under **Scopes**, click “Add or Remove Scopes”. (See [Choosing Scopes](#choosing-scopes) below for which scope(s) to add.)
5. Click **Save and Continue**.
6. Add your testing email under **Test Users** and save.

## Step 4: Create OAuth 2.0 Client ID and Secret

1. Go to **APIs & Services > Credentials**.
2. Click **+ CREATE CREDENTIALS > OAuth client ID**.
3. For **Application type**, select **Web application** (if your app is web-based) or **Desktop app** for testing/CLI dev.
4. Add an **authorized redirect URI**: this should match the URI OpenInt expects (e.g. `https://YOUR_APP_DOMAIN.com/oauth/callback/google-docs`). See your OpenInt config or [data flow docs](/connect/data-flow) for the exact URI to use.
5. Click **Create**.
6. **Copy the Client ID and Client Secret**—you’ll need to enter these into your OpenInt or app configuration.

⚠️ **Note:** Don't forget to "Publish" the OAuth consent screen if your integration is for production or accessed by users outside of test accounts!

## Step 6: Configure OpenInt with Your OAuth Credentials

You will need to add your Google OAuth Client ID and Secret to your OpenInt connector configuration. Example config:

```ts
// Example snippet (TypeScript/JSON)
{
  "provider": "google-docs",
  "client_id": "YOUR_CLIENT_ID.apps.googleusercontent.com",
  "client_secret": "YOUR_CLIENT_SECRET",
  "scope": "https://www.googleapis.com/auth/documents.currentonly",
  "redirect_uri": "https://YOUR_APP_DOMAIN.com/oauth/callback/google-docs"
}
```

Refer to the [OpenInt connector config docs](/core-concepts-v0/connector-config) for more details on available configuration options. For data flow and embedding, see [Data Flow](/connect/data-flow) and [Embed](/connect/embed).

## Common Setup Mistakes

- **Redirect URI mismatch**: The URI registered in Google Cloud must exactly match the one used in OpenInt and your application. Double-check for typos and trailing slashes.
- **API not enabled**: Ensure you have enabled “Google Docs API” (and “Google Drive API” if using Drive-related scopes) in your Google Cloud Project.
- **Test users not added**: For apps in testing, add your user email under Test Users on the OAuth consent screen.
- **Not requesting offline access**: To enable background token refresh, ensure `access_type=offline` is sent in the auth request (OpenInt’s default config manages this).
- **Unsupported scope**: Only use scopes documented above as supported by OpenInt. Unsupported scopes will result in failure to authorize or insufficient permissions.
