# Integrating Google Docs with OpenInt

Integrate Google Docs seamlessly into your application via OpenInt's secure, user-friendly connector. This guide walks you through every step, from setting up a Google Cloud OAuth2 client, through configuring the right scopes, to launching authentication flows in your app. 

> âš¡ **Current support:** As of this release, OpenInt supports the [`documents.currentonly`](https://developers.google.com/workspace/docs/api/auth#docs-scopes) scope for Google Docs. See details below on usage and roadmap for expanded scope support.

---

## Prerequisites

- Access to the [Google Cloud Console](https://console.cloud.google.com/) with project creation permissions
- OpenInt administrator access (sign up [here](https://app.openint.dev/dashboard/sign-up))
- Ability to configure OAuth consent and credentials

# 3. Configuring OpenInt with Your Credentials

Once you have your Google OAuth Client ID and Secret, plug them into the Connectors section of your OpenInt dashboard (see [Connector Configuration guide](/core-concepts-v0/connector-config)).

You will need to set:
- **Client ID**: from Google Cloud credentials
- **Client Secret**: from Google Cloud credentials
- **Redirect URI**: <code>https://app.openint.dev/oauth/callback</code>
- **Scope**: <code>https://www.googleapis.com/auth/documents.currentonly</code>

Example configuration:

```json
{
  "provider": "google-docs",
  "client_id": "YOUR_GOOGLE_CLIENT_ID",
  "client_secret": "YOUR_GOOGLE_CLIENT_SECRET",
  "scopes": ["https://www.googleapis.com/auth/documents.currentonly"],
  "redirect_uri": "https://app.openint.dev/oauth/callback"
}
```

# 4. Embedding the OpenInt Connect Widget

You can trigger the Google Docs OAuth flow in your application UI using the Connect Widget:

```jsx

```

# Common Mistakes & Troubleshooting

- **Redirect URI mismatch:** Ensure the URI in your Google Cloud credentials exactly matches <code>https://app.openint.dev/oauth/callback</code>.
- **Consent screen not published/test user not added:** If using "Testing" mode, add your email as a test user.
- **Scope errors:** Only <code>documents.currentonly</code> is supported by OpenInt right now. Other scopes will silently fail or be rejected by Google/OpenInt.
- **Missing API enablement:** Double-check that the Google Docs API is enabled in your Cloud project.
- **User denied consent:** Users must grant permission for your requested scope.
- **Unverified app warnings:** If you use sensitive scopes or havenâ€™t fully configured branding, Google may present the "unverified app" screen. For most integrations, do <u>not</u> request verification unless youâ€™re ready for security review.

For more troubleshooting tips, see the [OpenInt FAQ](/connect/faq) or [Google's OAuth troubleshooting page](https://developers.google.com/identity/protocols/oauth2#troubleshooting).
