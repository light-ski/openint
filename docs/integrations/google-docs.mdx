# Integrating Google Docs with OpenInt

This guide helps developers integrate Google Docs into their applications using OpenInt’s prebuilt connectors. You’ll learn how to register your Google Cloud OAuth credentials, select and configure the right scopes, set up the OpenInt Connect flow, and troubleshoot common pitfalls.

> ℹ️ **What You’ll Achieve:** Enable users to securely link their Google Docs account to your app via OpenInt’s widget, supporting document sync, read, and collaboration scenarios.

---

## Prerequisites

- A [Google Cloud Console](https://console.cloud.google.com/) account or access to your organization’s Google Cloud project
- Your OpenInt developer account ([sign up](https://app.openint.dev/dashboard/sign-up))
- Familiarity with [embedding OpenInt Connect](/connect/embed)

---

## Step 1: Create Google Cloud OAuth Credentials

To allow users to authenticate with Google Docs, your app needs OAuth 2.0 credentials from a Google Cloud project. Here’s the process, similar to the style outlined [here](https://www.2brightsparks.com/resources/articles/gdriveclientid.html):

<Steps>
  <Step title="Go to Google Cloud Console">
    Visit the [Google API Console](https://console.cloud.google.com/) and sign in.<br />
    Select an existing project or click **New Project** to create one.
  </Step>
  <Step title="Enable the Google Docs API">
    In the sidebar, go to **APIs & Services > Library**.<br />
    Search for "Google Docs API" and click **Enable**.
  </Step>
  <Step title="Configure OAuth Consent Screen">
    - In the sidebar, navigate to **APIs & Services > OAuth consent screen**.<br />
    - Select **External** user type for public apps.<br />
    - Fill in the required fields (App Name, User Support Email, Developer Contact Information).<br />
    - Add your company’s Google Workspace users or intended testers under **Test Users** (if app is not published).
  </Step>
  <Step title="Create OAuth Client ID">
    - Go to **APIs & Services > Credentials**.<br />
    - Click **Create Credentials > OAuth client ID**.<br />
    - Choose **Web application**.<br />
    - Name your OAuth client (e.g., `OpenInt Google Docs Connector`).<br />
    - Under **Authorized redirect URIs**, add:  
      `https://connect.openint.dev/oauth/callback`  
      (If self-hosted, add your domain’s equivalent callback.)
    - Click **Create** and note your **Client ID** and **Client Secret**.
  </Step>
  <Step title="(Optional) Publish App or Add Testers">
    - For restricted or production deployments, publish your app via the **OAuth consent screen** panel.<br />
    - **Do not click “Prepare for Verification” unless requested by Google.**
  </Step>
</Steps>

---

## Step 2: Configure Google Docs Integration in OpenInt

With your **client ID** and **client secret**, configure the Google Docs connector in the OpenInt dashboard:

<Steps>
  <Step title="Go to Connector Configs in OpenInt">
    - Log in to your [OpenInt dashboard](https://app.openint.dev/dashboard/).
    - Open **Connector Configs** in the sidebar.
    - Select **Google Docs** from the list of connectors.
  </Step>
  <Step title="Enter OAuth Credentials">
    - Paste the **client ID** and **client secret** you obtained above.
    - Ensure you select the Google Docs integration stage (usually `ga`/General Availability).
  </Step>
  <Step title="Choose Required Scopes">
    - Review the available scopes (see below for matrix) and select those your app needs.
    - We recommend starting with the least-permission scopes (see [Google's API Scopes docs](https://developers.google.com/workspace/docs/api/auth)).
  </Step>
  <Step title="Save and Verify">
    - Click **Create** or **Save**.
    - Confirm the connector appears in your OpenInt dashboard (the status should be **Active**).
  </Step>
</Steps>

## Step 3: Embed or Launch the Connect Flow

Embed OpenInt’s Connect widget in your app using your backend-generated customer token to enable OAuth linking. Here’s an example:

<CodeGroup>
```javascript React
import { ConnectEmbed } from '@openint/connect'

// token generated securely on backend
return <ConnectEmbed token={token} />
```

```javascript VanillaJS
import { connect } from '@openint/connect'
connect.embed({
  token: 'YOUR_CUSTOMER_TOKEN',
  containerRef: '#openint-connect-container',
})
```
</CodeGroup>

## Common Pitfalls and Troubleshooting

- **Redirect URI Mismatch**: Ensure the OAuth redirect URI exactly matches `https://connect.openint.dev/oauth/callback` (or your self-hosted equivalent).
- **“access_type=offline” Not Set**: OpenInt sets `access_type=offline` for refresh token support. If users must re-auth too frequently, check the consent screen params.
- **Out-of-date Credentials**: If you rotate OAuth secrets or change credential settings, update them immediately in OpenInt.
- **Missing Scopes**: Only scopes listed above are presently supported by OpenInt. Request others via support.
- **User Not Whitelisted**: If app is in testing, the authenticating user must be added as a test user on the OAuth consent screen.
- **Do Not Prepare for Verification Unless Required**: Avoid unnecessary delays; only proceed if requested by Google.
