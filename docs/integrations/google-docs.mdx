# Google Docs Integration (OAuth2)

Integrate **Google Docs** into your application with OpenInt, leveraging secure [OAuth 2.0](https://developers.google.com/identity/protocols/oauth2) authorization and a configurable permissions model. This guide walks you through registering your Google app, obtaining credentials, and connecting Google Docs to your organization using OpenInt.

> Refer to the official [Google Docs API Auth page](https://developers.google.com/workspace/docs/api/auth) for comprehensive details on OAuth setup and supported scopes.

---

## 1. Register an OAuth Application with Google

Follow these steps to create OAuth credentials for Google Docs. This is similar to the [2BrightSparks GDrive OAuth Guide](https://www.2brightsparks.com/resources/articles/gdriveclientid.html):

### 1.1. Create a Google Cloud Project
1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. At the top, open the project dropdown and choose **New Project**.
3. Enter a project name. Click **Create**.
4. In the dropdown, select your new project.

### 1.2. Enable the Google Docs API
1. In the left sidebar, navigate to **APIs & Services > Library** or [click here](https://console.cloud.google.com/apis/library).
2. Search for **Google Docs API** and click on it.
3. Click **Enable**.

### 1.3. Configure the OAuth Consent Screen
1. In the left sidebar, click **APIs & Services > OAuth consent screen** or [use this link](https://console.cloud.google.com/apis/credentials/consent).
2. Choose **External** (for most integrations) and click **Create**.
3. Fill in required app information (name, support email, etc.).
4. On the **Scopes** page, click **Add or Remove Scopes** — you will select required API scopes in a later step, but at a minimum: `https://www.googleapis.com/auth/documents.currentonly`.
5. (External apps) Add test user emails for development; you may add up to 100 prior to verification.
6. Complete the setup wizard and save.

### 1.4. Create OAuth Client ID & Secret
1. In **APIs & Services > Credentials**, click **Create Credentials > OAuth client ID**.
2. Choose **Web application** as the type.
3. Name your credential (e.g., `OpenInt Google Docs`).
4. For **Authorized redirect URIs**, add:
   - Your app's OpenInt callback URL (e.g., `https://yourapp.com/oauth2/callback`)
   - For local dev: `http://localhost:3000/oauth2/callback`
5. Click **Create**.
6. Copy the **Client ID** and **Client Secret** from the confirmation dialog. Store securely. Do not commit secrets to source code.

For advanced details, see [official Google guide](https://developers.google.com/identity/protocols/oauth2#creatingcred).

## 3. Configuring OpenInt for Google Docs

Add your credentials and specify `documents.currentonly` as the scope in your OpenInt connector configuration. Example:

```ts
{
  provider: 'google-docs',
  clientId: 'YOUR_GOOGLE_CLIENT_ID',
  clientSecret: 'YOUR_GOOGLE_CLIENT_SECRET',
  redirectUri: 'https://yourapp.com/oauth2/callback',
  scopes: ['https://www.googleapis.com/auth/documents.currentonly']
}
```

> Ensure your `redirectUri` matches exactly with what you registered in Google Cloud Console.

---

## 4. Happy Path OAuth Flow

1. User initiates connection between your app and Google Docs via OpenInt.
2. User is redirected to Google, consents to the requested scopes.
3. Google returns an authorization code to your registered callback (`redirectUri`).
4. OpenInt completes the OAuth flow, retrieving tokens and securely storing them for API use.
5. Your app can now interact with Google Docs (subject to permitted scope).

## 5. Troubleshooting & Common Pitfalls

- **Invalid redirect URI**: Ensure all possible OpenInt callback URLs are listed in your Google OAuth client configuration—*exact matches only*.
- **Missing APIs enabled**: If you see `access_denied` or `API not enabled` errors, confirm Google Docs API is enabled for your Cloud project.
- **OAuth consent screen not completed**: Complete all required fields and scope declarations.
- **Not listed as test user**: During development, add your Google account as a test user in the consent screen or you will be blocked.
- **Scope errors**: Double-check that only the supported scope (`documents.currentonly`) is requested.
- **Tokens expire quickly in test mode**: When app is unverified, refresh tokens last only 7 days; see [app verification process](https://support.google.com/cloud/answer/13461325?hl=en).

For more Google API troubleshooting tips, see the [official troubleshooting guide](https://support.google.com/cloud/answer/9110914).
