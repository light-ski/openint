# Integrate Google Docs with OpenInt

Connect your application to Google Docs using OpenInt‚Äôs secure OAuth2 workflow. This guide walks you through setting up your Google Cloud project, configuring OAuth credentials, and connecting Google Docs via OpenInt‚Äîenabling seamless syncing for your users.

> 
> Engineers at Company A or similar SaaS developers looking to integrate Google Docs access into their app using OpenInt‚Äôs native connectors.

## Prerequisites

- Access to [Google Cloud Console](https://console.cloud.google.com/)
- An OpenInt account ([sign up here](https://app.openint.dev/dashboard/sign-up))
- Admin access to configure integrations in OpenInt

---

## 1. Configure a Google Cloud Project for OAuth

Follow these steps to prepare your Google project and get the OAuth credentials needed for OpenInt:

<Steps>
  <Step title="Create a New Project">
    1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
    2. Click the project select menu (top bar), then "New Project".
    3. Give your project a name (e.g., `OpenInt Docs Integration`) and click **Create**.
  </Step>
  <Step title="Enable the Google Docs API">
    1. With your new project selected, go to **APIs & Services ‚Üí Library**.
    2. Search for "Google Docs API" and click **Enable**.
  </Step>
  <Step title="Set Up OAuth Consent Screen">
    1. Navigate to **APIs & Services ‚Üí OAuth consent screen**.
    2. Choose **External** if app users are outside your organization, then **Create**.
    3. Fill out the app information (name, support email). Use accurate info for organizational use.
    4. Add the required scopes (see below for supported scopes).
    5. *You may use test users for development until your app is verified by Google.*
  </Step>
  <Step title="Create OAuth Client Credentials">
    1. Go to **APIs & Services ‚Üí Credentials**.
    2. Click **Create Credentials ‚Üí OAuth client ID**.
    3. Choose **Web application**.
    4. Under **Authorized redirect URIs**, add:  
       `https://connect.openint.dev/oauth/callback`  
       (or your custom OpenInt Connect instance URI)
    5. Click **Create**. Save the **Client ID** and **Client Secret**‚Äîyou'll need these in OpenInt.
    <Frame>
      <img src="/images/gcloud-oauth-client.png" alt="Google Cloud OAuth Client Creation" />
    </Frame>
  </Step>
</Steps>

[Reference: Official Google OAuth guide](https://www.2brightsparks.com/resources/articles/gdriveclientid.html)

## 2. Add Google Docs Integration in OpenInt

Set up the Google Docs connector using your OAuth credentials:

<Steps>
  <Step title="Access Connector Configs in OpenInt">
    1. Log into your [OpenInt dashboard](https://app.openint.dev/dashboard/).
    2. Go to **Connector Configs** from the sidebar.
    <Frame>
      <img src="/images/dashboard.png" alt="OpenInt Console - Connector Configs" />
    </Frame>
  </Step>
  <Step title="Configure the Google Docs Connector">
    1. Click **Add Connector Config**.
    2. Choose **Google Docs** from the list.
    3. Enter your **Client ID** and **Client Secret** from your Google Cloud project.
    4. Ensure the redirect URI matches the one registered in Google Cloud.
    5. Select the scope: <br /><code>https://www.googleapis.com/auth/documents.currentonly</code>
    <Frame>
      <img src="/images/connector-form.png" alt="Connector Form - Google Docs" />
    </Frame>
    > ‚ÑπÔ∏è At present, OpenInt supports the limited `documents.currentonly` scope (access only to the currently open Docs document). Broader scopes are not yet available‚Äîsee table below.
  </Step>
  <Step title="Save and Test">
    1. Click **Create**.
    2. Your connector appears in the table. You‚Äôre now ready to authenticate users and sync Google Docs data.
    <Frame>
      <img src="/images/saved-connector.png" alt="Google Docs Connector Saved" />
    </Frame>
  </Step>
</Steps>

## 5. Common Pitfalls & Troubleshooting

- **Mismatched Redirect URIs:** Ensure the redirect URI in Google matches exactly what you configure in OpenInt. Missing `/oauth/callback` or typos are common causes for failed OAuth logins.
- **OAuth Consent Screen Not Verified:** For production use or when accessing data from users beyond your own org, Google may require you to submit your app for verification if using broader scopes (currently not supported in OpenInt for Google Docs).
- **Missing or Incorrect Scopes:** If you request a scope in Google not supported by OpenInt, auth will fail‚Äîuse only `documents.currentonly` for now.
- **Revoked/Expired Credentials:** If auth tokens expire or are revoked, users must re-authorize via OpenInt Connect.
- **Rate Limiting:** Hitting Google Drive API quotas is rare but can occur with large-scale syncs; configure your Google project and review quotas if you see quota errors.

For data model details and more about connectors, see [Data Model & Flows](/connect/data-flow) and [Connector Config](/core-concepts-v0/connector-config).
